# 📝 公司程式 AI 需求規格

---

## 🏠 首頁（Home Page）

### 🔴 重要需求

#### 1. 側邊欄 Logo 設計
- 左側導覽欄位之 Logo 圖示需使用較適當的圖片。
- 圖片來源由使用者提供多個候選選項。
- 系統僅可從指定選項中選擇，不可自行生成或替換未提供之圖片。

#### 2. 側邊欄功能
- 側邊欄除了原有選項（Home、Agent Store、線上圖書館）外，新增 **設定** 選項。
- **設定** 選項包含三個子設定：
  1. **公告欄設定**  
  2. **Agent 權限設定**  
  3. **圖書館設定**

#### 3. 公告欄互動行為
- 公告欄中的每一則公告需支援點擊操作。
- 使用者點擊公告後，需以彈出視窗（Modal / Dialog）方式顯示公告完整內容。

---

## 🔴 公告彈出視窗需求

#### 1. 公告主旨
- 彈出視窗需顯示公告欄中對應的主旨文字（例如：海外規範更版）。

#### 2. 公告內容區
- 內容主要以 **英文** 顯示為主。
- 預留空間：約 **150 個英文字符** 以容納公告內容。

#### 3. 檔案連結區
- 下方顯示公告相關檔案的 **封面圖片**。
- 設置一個 **Logo 按鈕**，點擊後可連結至該檔案的 **PDF 預覽**。

---

## 🤖 Agent Store

### 🔴 重要需求

#### 1. 主要選項
- 點擊 **Agent Store** 後，需顯示兩個選項：
  1. **對話 (Chat)**
  2. **對話歷史 (Chat History)**

#### 2. 對話 (Chat) 功能
- 選擇「對話」後，右側顯示 **使用者可用的 Agent 列表**。
- 類似 **App Store** 介面，方便快速選擇與啟動 Agent。
- 需顯示 Agent 名稱、使用模型、狀態，以及「對話」按鈕。

#### 3. 對話歷史 (Chat History) 功能
- 選擇「對話歷史」後，顯示使用者先前所有對話記錄。
- 對話歷史應可按時間或 Agent 進行排序或瀏覽。

---

## 📚 線上圖書館（Online Library）

### 🔴 重要需求

#### 1. 卡片概覽（首頁）
- 每張卡片代表 **一個館名**（例如：Cloud Archite）。
- 若館名卡片過多，使用 **滑動瀏覽**（水平或垂直滾動）。
- 卡片內文件列表以 **條列式呈現**：
  - 每行顯示一個 **文件名稱**  
  - 若文件過多，使用 **分頁功能**瀏覽文件（例如每頁 4~5 個文件）
- **互動**：
  - 點擊任意 **文件名稱** → 展開該文件
  - 顯示 **PDF 預覽** 並提供 **下載按鈕**

#### 2. 點擊館名展開
- 點擊 **館名** 後，卡片內文件轉為 **卡片式呈現**：
  - 每個文件顯示：
    - **圖示**  
    - **檔名**  
    - **簡介**
- 展開後若文件過多，使用 **滾動瀏覽**（而非分頁）。
- **互動**：
  - 點擊任意 **文件卡片** → 展開該文件
  - 顯示 **PDF 預覽** 並提供 **下載按鈕**

## ⚙️ 側邊欄設定

### 🔴 公告欄設定

#### 1. 新增公告
- 使用者可以新增公告。
- 可編輯欄位：
  - **主旨**（公告標題）  
  - **內容**（以英文為主，預留約 150 字元空間）  
  - **附件檔案**（可選擇是否夾帶 PDF 檔）
- 可選擇 **發布公告**，使公告立即出現在首頁公告欄。

#### 2. 編輯公告
- 使用者可編輯已存在公告：
  - **修改主旨**  
  - **修改內容**  
  - **新增或移除附件檔案**  
  - **發布或取消發布**

#### 3. 刪除公告
- 使用者可刪除不需要的公告。
- 刪除操作需有確認提示，避免誤刪。

#### 4. 查閱公告歷史
- 提供公告歷史列表，顯示過去所有公告。
- 可按時間排序或搜尋關鍵字。

### 🔴 Agent 權限設定

#### 1. 指派 Agent 給使用者
- 系統管理者可將 **特定 Agent** 指派給最多 **50 位使用者**授權使用。
- 指派後，被授權使用者才可在 **Agent Store** 中使用該 Agent。

#### 2. 指定 Agent 上架狀態
- 系統管理者可控制哪些 Agent 可 **發布於 Agent Store**。
- 未被發布的 Agent 無法被一般使用者看到或使用。

### 🔴 圖書館設定

#### 1. 上傳知識文件
- 系統管理者 (Admin) 可上傳 **靜態知識文件** 至線上圖書館。
- 上傳時需填寫必要欄位，例如文件名稱、館名、簡介等。

#### 2. 刪除知識文件
- 系統管理者可刪除已上傳的知識文件。
- 刪除操作需有確認提示，避免誤刪。

#### 3. 指定知識文件權限
- 系統管理者可設定每個知識文件的使用權限（Option）。
- 權限設定影響哪些使用者可在線上圖書館看到或預覽該文件。
